Getting Started With the JavaScript SDK

Pre-release Documentation

Although Carbon LDP is not yet released, we're publishing documentation as it is prepared so that you can preview what's in store.

Assumptions

While we try to make the documentation as friendly as possible, it is about CarbonLDP, so in order to keep the focus on it we need to set a baseline.
To be able to understand our documentation, a working knowledge on the following topics is needed:

  • HTML, CSS, JavaScript: After all, we'll be covering Carbon's JAVASCRIPT SDK. You can find documentation on these topics at the Mozilla Developer Network.
  • Promises: A standard-based way of handling asynchronous JavaScript logic. Mozilla Developer Network has documentation about them here.

There are also some topics that are not strictly required, but Carbon leverages them to enhance the coding experience. Knowledge on this topics is highly recommended:

  • TypeScript: Carbon's JavaScript SDK is written in TypeScript, a JavaScript superset language, maintained by Microsoft. You can find documentation here.
  • Package Managers: Carbon recommends JSPM
  • ES2015 Modules: Working with Carbon is way smoother when using ES2015 modules. They let you "import" a file from another one so you can modularize your application. There's more information here.

Installing the SDK

Installing the SDK depends on what package manager you are using:

Package manager?

If you are not using a package manager you can clone the source code with the following command:

git clone https://github.com/CarbonLDP/CarbonLDP-JS-SDK.git

It already includes a pre-compiled version of the source code so you can just jump right in.

npm 
npm install carbonldp --save
jspm 

jspm install npm:carbonldp
# We also need to install one of carbonldp's dependencies
jspm install npm:jsonld

After it has been installed the way you'll include it depends on your development environment:

Browser - Global Window Object

The SDK comes with a bundled version that you can import directly into your html:


<script src="--sdk-directory--/dist/bundles/Carbon.sfx.js"></script>

After that, a variable named Carbon will be accessible in the global scope. This variable will serve as both a class constructor and a namespace. It's usage as a class constructor is the same as when using ES2015 Modules, but its namespace nature is unique to the Global approach.

Carbon variable as a namespace

The Carbon object chains together all the Carbon modules together, so you can access them even if you are using the Global approach. Each time you see in an example something like: 


import * as Errors from "carbonldp/HTTP/Errors";

You can take the portion after the carbonldp/, replace the /s with . and append that to the Carbon namespace to access the same module. For example, accessing that module using the global namespace would be: 


Carbon.HTTP.Errors
Browser - ES2015 Modules

To be able to use ES2015 modules a transpiler like Babel, Traceur or TypeScript needs to be used. This documentation won't cover the details of setting one up, but we recommend using JSPM as a package manager, configured to use TypeScript as a transpiler. That way you can use ES2015 features and bundle your dependencies together!

Each module in Carbon will export at least one named export. Some of the modules will also export a default export containing the "main" export of the module. That export will also be available as the named export Class. For example: 


// This:
import Document from "carbonldp/Document";
let myDocument:Document = new Document();

// Is the same as this:
import * as Document from "carbonldp/Document";
let myDocument:Document.Class = new Document.Class();

We recommend not using default exports, but we provide them in case you are only using the "main" export of a file.

The semantics behind Carbon's modules are based on the usage of the import all statement. They'll likely change in the future, but because modern IDEs lack proper auto-import functionality (WebStorm is getting closer...), importing everything of a module is faster than doing it selectively. Still, you can use selective imports but you'll likely need to rename the import variables like: 


import { Factory as DocumentFactory, Class as Document } from "carbonldp/Document";

For each folder inside the Carbon source, there is a module at the same level with the same name that exports everything contained by the folder. Example: 


import * as Errors from "carbonldp/HTTP/Errors";
import * as Request from "carbonldp/HTTP/Request";

import * as HTTP from "carbonldp/HTTP";

HTTP.Errors === Errors; // true
HTTP.Request === Request; // true

Creating contexts

Now that you have your brand new app (If you don't have your app yet, please create it! (link) ). Let's start with some basic things you can do, like creating documents, creating parent-children and other types of links, saving documents etc.

Before you can manipulate Carbon objects, you need to create a Carbon platform context. This context will provide a reference to the Carbon platform instance you are working with. (more about Carbon context's can be found here)

After you create a Carbon platform context, you need an Application context that references the application on which you will be working on. The app context is retrieve from a previously created application

TypeScript JavaScript 

// ...
import * as App from "carbonldp/App";
import Carbon from "carbonldp/Carbon";

let carbon:Carbon = new Carbon(); //platform context

let myAppContext:AppContext;
carbon.apps.get( "your-app-slug-here/" ).then( ( appContext:App.Context ) => {
	myAppContext = appContext;
}).catch( console.error );     //retrieving you app context
							

import Carbon from "carbonldp/Carbon";

let carbon = new Carbon(); //platform context
let myAppContext;

carbon.apps.get( "your-app-slug-here/" ).then( ( appContext ) => {
	myAppContext = appContext;
}).catch( console.error );     //retrieving you app context

The apps.get() method returns a Promise that when resolved, will give us the App Context we need

Retrieving a Document

Now with our App Context we can start working with our App's data. We'll retrieve our root Document which is the root of all of our Document nodes.

TypeScript JavaScript 


import * Response from "carbonldp/Document";
import * Document from "carbonldp/Document";

let rootDocument:Document.Class;

myAppContext.documents.get( "/" ).then(
	( [ document, response ]:[ Document.Class, Response.Class ]) => {
		rootDocument = document;
}).catch( console.error );


import * as Document from "carbonldp/Document";
import * as Response from "carbonldp/HTTP/Response";
import * as App from "carbonldp/App";
import Carbon from "carbonldp/Carbon";

let carbon:Carbon = new Carbon(); //platform context
let myAppContext:AppContext;
let rootDocument:Document.Class;

carbon.apps.get( "your-app-slug-here/" ).then( ( appContext:App.Context ) => {
	myAppContext = appContext; //retrieving you app context
	return	myAppContext.documents.get( "/" );
}).then( ( [ document, response ]:[ Document.Class, Response.Class ] ) => {
	rootDocument = document;
}).catch( console.error );
							

// ...

	rootDocument = document;
});

The documents.get() method accepts a relative URI. That relative URI is going to be resolved to an absolute URI using the context in which we are working on (in this case appContext). To see to what URI it'll resolve you can execute appContext.resolve( "/" ).

Creating a Document

The root Document that you just retrieved can be seen as a the root folder of your whole structure. It is not recommended to modify it. Instead you should create children Documents that will hold your data. You can start by creating Documents in memory, that will have to be persisted later, or you can directly create a persisted Document.

To create an in memory document you can follow two paths. You can create a document in memory with Document.Factory.create(), and then you can define as many properties as you wish in it. Or you can create a Document from a javascript object using Document.Factory.createFrom(). In both cases, the document created will not be a persisted Document, until you specifically persist it. The creation of a persisted document is discussed in the next section Persisting a Document.

TypeScript JavaScript 


import * as Document from "carbonldp/Document";

//Create an empty Document
let inMemoryDocument:Document.Class = Document.Factory.create();

// We can assign any property we want to it
inMemoryDocument.someProperty = "Property value";
inMemoryDocument.anotherProperty = 123;

//Create a Document from an object
let inMemoryOtherDocument:Document.Class = Document.Factory.createFrom( {
	someProperty: "Property value",
	anotherProperty: 123
} );
							


import * as Document from "carbonldp/Document";

let inMemoryDocument = Document.Factory.create();

// We can assign any property we want to it
inMemoryDocument.someProperty = "Property value";
inMemoryDocument.anotherProperty = 123;

let inMemoryOtherDocument = Document.Factory.createFrom({
	someProperty: "Property value",
	anotherProperty: 123
});

In the past example you just created a simple Document with some properties, the Documents , however, can have properties, fragments and named fragments. You can create a Document with fragments from an object, or you can create fragments in a previously created Document. In the next sample you will see both methods

TypeScript JavaScript 

import * as Document from "carbonldp/Document";
import * as Fragment from "carbonldp/Fragment";
import * as NamedFragment from "carbonldp/NamedFragment";

let inMemoryDocument:Document.Class = Document.Factory.create();
inMemoryDocument.someProperty = "Property value";
inMemoryDocument.anotherProperty = 123;

//Create fragments in an in memory Document
let fragment:Fragment.Class = inMemoryDocument.createFragment( {
	someProperty: "hello"
} );

let namedFragment:NamedFragment.Class = inMemoryDocument.createNamedFragment( "my-slug", {
	someOtherProperty: "good-bye"
} );

//Create an in memory Document with fragments from an object
let inMemoryOtherDocument:Document.Class = Document.Factory.createFrom( {
	someProperty: "Property value",
	anotherProperty: 123,
	data: [
		// Fragment
		{
			someProperty: "hello"
		},
		// Named Fragment
		{
			id: "#my-slug",
			someOtherProperty: "good-bye"
		}
	]
} );

Persisting a Document

A persisted Document is a Document ...that has been saved in your platform ???... All documents are persisted through the method appContext.documents.createChild( "parent-id/" ,[ "child-id/" ]... ). The method is named createChild() since all the documents must have a parent Document, being the root Document the first parent in your application, as mentioned in Retrieving a document, the root Document id is "/".

You can persist an in memory Document or you can directly create a persisted Document. This next sample code shows how to persist a new Document or an in memory Document.

TypeScript JavaScript 

//...
import * as Pointer from "carbonldp/Pointer";
import * as Response from "carbonldp/HTTP/Response";

//...

myAppContext.documents.createChild( "parent-id/", myDocument ).then(
	( [ documentPointer, response ]:[ Pointer.Class, Response.Class ] ) => {
		console.log( documentPointer.id); //your persisted Doc id
});

//Persisting an object as a child Document of the root Document
myAppContext.documents.createChild( "/", {
		name: "Miguel",
		data: [
			// Fragment
			{
				someProperty: "hello"
			},
			// Named Fragment
			{
				id: "#my-slug",
				someOtherProperty: "good-bye"
			}
		]
}).then(( [ documentPointer, response ]:[ Pointer.Class, Response.Class ] ) => {
	console.log( documentPointer.id); //your persisted Doc id
});

The documents.createChild() method return an unresolved Document Pointer to the child Document you just created. When the pointer is resolved you can have access to the Document's content.

You can provide an id to the Document you are creating. The id is known as the child-id and it accepts any string. Carbon takes the string as a suggestion for the id of the child Document you are creating, the string will be ultimately converted into an acceptable id that can be retrieve by using a document Pointer, this will be discussed in the section Document Pointers. The next sample code shows the method createChild used with the child-id parameter.

TypeScript JavaScript 

//...
import * as Pointer from "carbonldp/Pointer";
import * as Response from "carbonldp/HTTP/Response";
//...

//Persisting the object as a child of the root Document, with the id child-id
myAppContext.documents.createChild( "/", "child-id", {
	name: "Miguel",
	data: [
		// Fragment
		{
			someProperty: "hello"
		},
		// Named Fragment
		{
			id: "#my-slug",
			someOtherProperty: "good-bye"
		}
	]
} );

//Persist myDocument as a child of "parent-id/", with an id "my-new-child"
myAppContext.documents.createChild( "parent-id/", "my new child", myDocument ).then(
	( [ documentPointer, response ]:[ Pointer.Class, Response.Class ] ) => {
		 console.log( documentPointer.id); //the id of you newly created Document
	}
);

Resolving a Document Pointer

In the last section you created a new Document, and after it's creation you obtained a response in the form of a Document Pointer. The document Pointer is referencing the place where your Document is at, and before resolving it, the property documentPointer.id gives you the id of the Document it points at. To access the Document and the information it contains you need to resolve the Pointer.

In the next sample code you will see how to resolvea Document Pointer from a recently created Document. Every time a method return a Document Pointer it can be resolved in this way.

TypeScript JavaScript 

//...
import * as PersistedDocument from "carbonldp/PersistedDocument";
//...

let myDocumentPointer:Pointer.Class;

myDocumentPointer.resolve().then(
	( [ document, response ]:[ PersistedDocument.Class, Response.Class ] ) => {
		console.log( document.name ); // "your-name"
});



import * as App from "carbonldp/App";
import * as Document from "carbonldp/Document";
import * as PersistedDocument from "carbonldp/PersistedDocument";
import * as Pointer from "carbonldp/Pointer";
import * as Response from "carbonldp/HTTP/Response";
import Carbon from "carbonldp/Carbon";

let platformContext:Carbon = new Carbon();
let myAppContext:App.Context;

let myDocument:Document.Class = Document.Factory.create();

let myDocumentPointer:Pointer.Class;
let myPersistedDocument:PersistedDocument.Class;

myDocument.name = "your-name";
myDocument.lastName = "your-lastName";
myDocument.age = "your-age";

platformContext.apps.getContext( "your-app-slug-here/" ).then( ( appContext:App.Context ) => {
	myAppContext = appContext;
	return myAppContext.documents.createChild( "/", myDocument );

}).then( ( [ documentPointer, response ]:[ Pointer.Class, Response.Class ] ) => {
	return documentPointer.resolve();

}).then( ( [ document, response ]:[ PersistedDocument.Class, Response.Class ] ) => {
	console.log( document.name ); // "your-name"
	myPersistedDocument = document;
}).catch( console.error );

Modifying and Saving a Document

All the Documents in Carbon can be modified. However all the changes done to a Document, are done in memory. Which means that after modifying a Document, the changes need to be saved, for them to be persisted.

You retrieve a Document with it's id as describe in the section Retrieving a Document with the method appContext.documents.get(). If it is a newly created Doucment you can obtained it by resolving the Pointer obtained after it's creation documentPointer.resolve() as seen in the Resolving a Document Pointer . After, obtaining it, you can add, modify or delete as many properties as you wish. Finally, you can save the changes in the Document with the method document.save()

TypeScript JavaScript 
								
import * as App from "carbonldp/App";
import * as PersistedDocument from "carbonldp/PersistedDocument";
import * as Document from "carbonldp/Document";
import * as Response from "carbonldp/HTTP/Response";
import Carbon from "carbonldp/Carbon";

let platformContext:Carbon = new Carbon();
let myAppContext:App.Context;
let myDocument:Document.Class;

platformContext.apps.getAppContext( "your-app-slug/" ).then( ( appContext:App.Context ) =>{
	myAppContext = appContext;
	return myAppContext.documents.get( "your-document-id/" );

}).then( ( [ document, response ]:[ Document.Class, Response.Class ] ) => {
	myDocument = document;
	myDocument.newProperty = "You!";
	myDocument.anotherProperty = anotherDocument;
	delete myDocument.someOldProperty;

	return myDocument.save();
}).then( ( [ document, response ]:[ PersistedDocument.Class, Response.Class ] ) => {
	console.log( document );
}).catch( console.error() );
