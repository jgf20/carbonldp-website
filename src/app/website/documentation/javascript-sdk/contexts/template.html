What is a context?

A Context is a container that references the place where all the you platform's settings and your applications' data are stored. Through the contexts you can comunicate with your Carbon server to access the settings, authentication services and object Schema of your platform and/or applications. You can have different contexts with different settings for each application, or you can use the platform context for some global settings. Having contexts helps you simplify the programming process, the contexts help you resolve all the relative URIs that you use in your applications.

Types of contexts

In the server side carbon handles two spaces, where your data is stored. The SDK mirrors these spaces using contexts.

  • /platform/
  • /apps/

In the platform space you will find all the general settings for your applications, agents, roles among others. In the apps space you will find everything specific to your application.

Platform context

In the platform context you can set your domain, meaning that you can set the URL where your platform is at, this way your application is more or less independent to the location of carbon, e.g. if your platform changes locations you just need to set the new domain in your Platform context to access you applications.

Declare the platform context

To declare a platform context you need to import Carbon from the carbonldp/Carbon library you can instance a new Carbon object, this is you platform context. If you just declare it, you can see the default values the context has. e.g. the default domain is https://carbonldp.com/ 

import Carbon from "carbonldp/Carbon";

let carbonContext:Carbon = new Carbon();


carbonContext.resolve( "" ); // https://carbonldp.com/platform/
// Change a setting
carbonContext.setSetting( "domain", "dev.carbonldp.com" );

carbonContext.resolve( "" ); // https://dev.carbonldp.com/platform/

let anotherContext:Carbon = new Carbon();
anotherContext.resolve( "" ); // https://carbonldp.com/platform/

Access/Modify your platform context

In the last section we saw how to declare a platform context, now we will see how to costumize it.

You can set your domain


import Carbon from "carbonldp/Carbon";

let carbonContext:Carbon = new Carbon();

carbonContext.setSetting( "domain", "dev.carbonldp.com" );

carbonContext.resolve( "" ); // https://dev.carbonldp.com/platform/

Application context

Once you have your platform context declared, you can get your application context. The application context is created along side your application, you can not reference a application context of an application that is not yet created. If you haven't created a application, please do so! method example... 


import Carbon from "carbonldp/Carbon";

let carbonContext:Carbon = new Carbon();

carbonContext.apps.getAppContext( "test-app/" ).then(
	( appContext:App.Context ) => {
		appContext.resolve( "" ); // https://carbonldp.com/apps/test-app/
	}
);

Inheritance

All the settings of platform contexts are by default inherited by the application contexts, you can over write them in you application context. This allows you to manage different objects schema in your applications, or manage different authentication credentials.
