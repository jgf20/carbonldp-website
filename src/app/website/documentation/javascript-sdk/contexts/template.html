What is a context?

A Context is a container that references the places where all the platform's settings or where your applications' data is stored. Through the contexts you can comunicate with your Carbon server to access the settings, information, authentication services and object Schema of your platform and/or applications.

Types of contexts

On the server side carbon handles two spaces, where your data is stored.

  • Platform space /platform/
  • Apps space /apps/your-app/

The SDK allows you to define two types of contexts by mirroring the server side spaces

  • Platform context
  • Application context

Having contexts helps you simplify the programming process. When you create an application it is created in both spaces, each space contains specific information that will be discussed in the next sections.

Platform context

The platform context references, as we said before, the /platform/ space, this helps you access some general settings of your platform, e.g. you can specify your platform domain by setting the URL where your platform is at, this way your application is more or less independent to the location of carbon server. The platform context also allows you to access /platform/apps/ where you can find a list of all your applications, and in /platform/apps/your-app-name/ you will find some application specific information. For now we will just focus on how to declare your platform context and how to set it.

Declare the platform context

To declare a platform context you need to import Carbon from the carbonldp/Carbon library and instance a new Carbon object, this will be your platform context. You can get the default value the context has by resolving it. e.g. the default domain is https://carbonldp.com/

TypeScript JavaScript 

import Carbon from "carbonldp/Carbon";

let carbonContext:Carbon = new Carbon();

carbonContext.resolve( "" ); // https://carbonldp.com/platform/


								

import Carbon from "carbonldp/Carbon";

let carbonContext = new Carbon();

carbonContext.resolve( "" ); //https://carbonldp.com/platform/

Access/Modify your platform context

In the last section we saw how to declare a platform context, now we will see how to set a different domain in it.

TypeScript JavaScript 

import Carbon from "carbonldp/Carbon";

let carbonContext:Carbon = new Carbon();

carbonContext.setSetting( "domain", "dev.carbonldp.com" );

carbonContext.resolve( "" ); // https://dev.carbonldp.com/platform/

								

import Carbon from "carbonldp/Carbon";

let carbonContext = new Carbon();

carbonContext.setSetting( "domain", "dev.carbonldp.com" );

carbonContext.resolve( "" ); //https://dev.carbonldp.com/platform;

Application context

An application context allows you to access particular settings for the application it references. You can configure the roles and agents specific to the application, and you can see what information it contains.

Getting the App Context

Once you have your platform context declared, you can get your application context. The application context is created along side your application, you can not reference an application context if an application is not yet created. If you haven't created a application, please do so!

TypeScript JavaScript 

import Carbon from "carbonldp/Carbon";

let carbonContext:Carbon = new Carbon();

carbonContext.apps.getAppContext( "test-app/" ).then(
	( appContext:App.Context ) => {
		appContext.resolve( "" ); // https://carbonldp.com/apps/test-app/
	}
);

								

import Carbon from "carbonldp/Carbon";

let carbonContext = new Carbon();

carbonContext.apps.getAppContext( "test-app/" ).then(
	(appContext) =>{
		appContext.resolve( "" ); //https://carbonldp.com/apss/test-app/
	}
);

Inheritance

All the settings of platform contexts are, by default, inherited by the application contexts, as long as they are not overwritten. This allows you to manage different settings and/or objects schema in your applications, or manage different authentication credentials. This is important to remember as you delve deeper in Carbon, for now it is enough to keep in mind this property of the contexts.
